Accessibility

Swisscom is looking forward to reaching at least the AA Compliance for their experience. It is still an ongoing process and we will keep improving the accessibility. Therefore, we work with "Access for all" to evaluate the SDX style guide in order to get an AA Certification, and even more important, to prevent excluding anyone.

By making your website accessible, you are ensuring that all users, including people with disabilities, have a good user experience and are able to easily access your information.

Web accessibility includes all disabilities that affect access to the Web:

  • auditory
  • cognitive
  • neurological
  • physical
  • speech
  • visual
  • impairments due to age

Approximately 20% of the Swiss population is affected (10’000 blind, 325’000 visually impaired, 500’000 auditorily impaired).

Rules

To achieve an AA compliant website, it’s necessary to follow some rules:

  • It's always recommended to use native HTML instead of ARIA-labels.
  • Implement a semantically correct structure (H1 followed by H2, P used for text, …).
  • Keep in mind that all elements of a website should be focusable by using the keyboard only, in the desired order and have a visual focus.
  • Do not delete ARIA information from SDX examples. It ensures the right interpretation by screen readers.

WAI-ARIA (Web Accessibility Initiative – Accessible Rich Internet Applications)

Add semantics to elements that are not native HTML (e.g. Web Components):

  • Use them sparingly
  • Use them consistently
  • Avoid redundant output through screen readers

Focus

Use skip links as an initial focus on your page which enables the screen reader quickly access the most important content.

Usage

You have to check the whole page semantic structure and context. Please check your website by using a screen reader – see section Testing Tools.

By using the accessible components, it is not a guarantee that the website is also accessible. The SDX consumer should take the responsibility of the whole accessible experience.
